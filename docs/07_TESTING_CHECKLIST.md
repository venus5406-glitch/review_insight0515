# 07. 테스트 검증 체크리스트

## 목표
UI, 프론트엔드 기능, 백엔드 API, OpenAI 연동, Supabase 저장, 배포 환경을 검증한다.

## 1. UI 테스트

| 항목 | 기대 결과 | 상태 |
|---|---|---|
| 페이지 접속 | 메인 화면이 정상 표시됨 | 미확인 |
| Header 표시 | 서비스명과 메뉴가 표시됨 | 미확인 |
| 텍스트 입력창 표시 | textarea가 표시됨 | 미확인 |
| 감성분석 버튼 표시 | CTA 버튼이 표시됨 | 미확인 |
| 정보 카드 표시 | 긍정/부정/중립 카드가 표시됨 | 미확인 |
| 모바일 화면 | 레이아웃이 깨지지 않음 | 미확인 |
| 버튼 클릭 인터랙션 | `scale(0.95)` 효과가 적용됨 | 미확인 |

## 2. 입력값 검증 테스트

| 케이스 | 입력 | 기대 결과 |
|---|---|---|
| 빈 값 | `""` | API 요청 없음, 에러 메시지 표시 |
| 공백만 입력 | `"   "` | API 요청 없음, 에러 메시지 표시 |
| 정상 입력 | `"좋아요"` | API 요청 발생 |
| 1,000자 초과 | 1,001자 이상 | API 요청 없음, 길이 초과 메시지 표시 |

## 3. API 테스트

### 정상 요청
```bash
curl -X POST http://localhost:3000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"text":"오늘 서비스가 정말 만족스러웠어요."}'
```

기대 결과:
```json
{
  "success": true,
  "data": {
    "sentiment": "positive",
    "sentimentLabel": "긍정",
    "confidence": 90,
    "reason": "..."
  }
}
```

### 빈 값 요청
```bash
curl -X POST http://localhost:3000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"text":""}'
```

기대 결과:
```json
{
  "success": false,
  "message": "분석할 텍스트를 입력해주세요."
}
```

## 4. 감성 분석 결과 테스트

| 입력 문장 | 기대 감성 |
|---|---|
| 오늘 서비스가 정말 만족스러웠어요. | 긍정 |
| 배송이 너무 늦고 실망스러웠습니다. | 부정 |
| 오늘 오후 3시에 회의가 있습니다. | 중립 |

## 5. 모달 테스트

| 항목 | 기대 결과 |
|---|---|
| 분석 성공 | 모달 자동 표시 |
| 감성 결과 | 긍정/부정/중립 표시 |
| 신뢰도 | 백분율로 표시 |
| 구분선 | 신뢰도 아래 표시 |
| 분석 이유 | 2~3문장 표시 |
| 닫기 버튼 | 모달 닫힘 |
| 확인 버튼 | 모달 닫힘 |
| 바깥 영역 클릭 | 모달 닫힘 |
| ESC 키 | 모달 닫힘 |

## 6. Supabase 저장 테스트

| 항목 | 기대 결과 |
|---|---|
| 분석 성공 후 저장 | `sentiment_analyses` 테이블에 row 생성 |
| sentiment 값 | positive/negative/neutral 중 하나 |
| confidence 값 | 0~100 사이 정수 |
| reason 값 | 비어 있지 않음 |
| created_at 값 | 자동 생성 |

## 7. 에러 처리 테스트

| 상황 | 기대 결과 |
|---|---|
| OpenAI API Key 없음 | 서버 오류 메시지 반환 |
| OpenAI API 호출 실패 | 사용자 친화적 오류 메시지 표시 |
| Supabase 저장 실패 | 분석 결과는 사용자에게 표시됨 |
| 네트워크 단절 | 네트워크 오류 메시지 표시 |
| 잘못된 JSON 응답 | AI 응답 처리 오류 메시지 표시 |

## 8. 보안 테스트

| 항목 | 기대 결과 |
|---|---|
| OpenAI API Key | 브라우저에서 노출되지 않음 |
| Supabase Service Role Key | 브라우저에서 노출되지 않음 |
| `.env` 파일 | Git에 포함되지 않음 |
| Vercel 환경변수 | 프로젝트 Settings에 등록됨 |

## 완료 기준
- 모든 필수 테스트가 통과한다.
- 실패 케이스에서 사용자에게 명확한 메시지가 표시된다.
- 보안 키가 프론트엔드에 노출되지 않는다.
