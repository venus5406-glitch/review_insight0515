# 02. UI 구현 명세

## 목표
HTML, CSS, JavaScript를 사용하여 감성 분석 서비스의 화면을 구현한다.

## 구현 파일
```text
public/
├── index.html
├── css/
│   └── style.css
└── js/
    └── app.js
```

## 화면 구성
```text
Header
Hero Section
Input Card
Result Modal
Info Cards
Footer
```

## 1. Header

### 구성 요소
- 서비스명: AI Sentiment Analyzer
- 메뉴:
  - 서비스 소개
  - 사용 방법

### 요구사항
- 상단 고정은 선택사항이다.
- 배경은 흰색을 사용한다.
- 부드러운 nav shadow를 적용한다.

## 2. Hero Section

### 구성 요소
- 메인 제목
- 설명 문구
- 텍스트 입력 카드
- 감성분석 버튼

### 문구
```text
AI 감성 분석 서비스
텍스트를 입력하면 AI가 문장의 감성을 긍정, 부정, 중립으로 분석합니다.
```

### 디자인
- 배경색: `#f2f0eb`
- 최대 너비: 720px
- 중앙 정렬
- 상하 여백: 64px
- 제목 색상: `#006241`
- 본문 색상: `rgba(0,0,0,0.58)`

## 3. 텍스트 입력창

### 요구사항
- 여러 줄 입력 가능
- 최소 높이 180px
- 최대 1,000자
- Placeholder 제공
- 현재 입력 글자 수 표시 권장

### Placeholder
```text
분석하고 싶은 문장을 입력하세요. 예: 오늘 서비스가 정말 만족스러웠어요.
```

## 4. 감성분석 버튼

### 기본 상태
```text
감성분석
```

### 요청 중 상태
```text
분석 중...
```

### 요구사항
- 입력값이 없으면 API 호출 금지
- 요청 중에는 버튼 비활성화
- 클릭 시 `scale(0.95)` 적용

## 5. 결과 모달

### 표시 항목
- 제목: 분석 결과
- 감성 결과
- 신뢰도
- 구분선
- 분석 이유
- 닫기 버튼
- 확인 버튼

### 예시 구조
```text
분석 결과

긍정
신뢰도 92%

----------------------------

문장에 긍정적인 표현이 포함되어 있습니다.
전반적인 어조가 호의적이며 불만 표현이 없습니다.

확인
```

### 닫기 조건
- 닫기 버튼 클릭
- 확인 버튼 클릭
- 모달 바깥 영역 클릭
- ESC 키 입력

## 6. 정보 카드 섹션

### 카드 3개
1. 긍정 분석
2. 부정 분석
3. 중립 분석

### 카드 문구 예시

#### 긍정 분석
```text
만족, 기대, 칭찬, 호의적인 표현을 감지합니다.
```

#### 부정 분석
```text
불만, 실망, 분노, 비판적인 표현을 감지합니다.
```

#### 중립 분석
```text
감정 표현이 약하거나 정보 전달 중심의 문장을 분류합니다.
```

## 7. 반응형 기준

| 화면 크기 | 구현 기준 |
|---|---|
| 480px 미만 | 단일 컬럼, 버튼 full-width |
| 768px 미만 | 정보 카드 단일 컬럼 |
| 768px 이상 | 정보 카드 3컬럼 |
| 1024px 이상 | 전체 콘텐츠 최대 너비 960~1200px |

## 완료 기준
- 메인 화면이 정상 렌더링된다.
- 텍스트 입력이 가능하다.
- 감성분석 버튼이 존재한다.
- 결과 모달 UI가 구현되어 있다.
- 모바일 화면에서 레이아웃이 깨지지 않는다.
